# CLI Commands

This page provides a comprehensive reference for all commands available in the Zopio CLI.

## Core Commands

### init

Initialize a new Zopio project with the default structure.

```bash
zopio init [options]
```

**Options:**
- `-t, --template <template>` - Template to use (default: "default")
- `-l, --locale <locale>` - Default locale for internationalization (default: "en")

**Example:**
```bash
zopio init --locale tr
```

The init command creates a new Zopio project with the following structure:
- Basic Next.js application setup
- Internationalization configuration
- Default directory structure
- Package.json with required dependencies

### generate

Generate a new Zopio module.

```bash
zopio generate <type> <n> [options]
```

**Arguments:**
- `type` - Type of module to generate (core | addon | data | i18n)
- `n` - Name of the module

**Options:**
- `-d, --directory <directory>` - Custom directory for the module

**Example:**
```bash
# Generate an i18n module
zopio generate i18n user-messages

# Generate a core module
zopio generate core authentication
```

### component

Generate a new React component with optional internationalization support.

```bash
zopio component <n> [options]
```

**Arguments:**
- `n` - Component name

**Options:**
- `-i, --i18n` - Add internationalization support
- `-p, --path <path>` - Custom path for the component (default: "components")

**Example:**
```bash
zopio component UserProfile --i18n
```

This will generate a component with built-in internationalization support using next-intl.

### i18n

Manage internationalization for your Zopio project.

```bash
zopio i18n [options]
```

**Options:**
- `-a, --add <locale>` - Add a new locale
- `-l, --list` - List all available locales
- `-e, --extract` - Extract translation keys from project
- `-c, --create <namespace>` - Create a new translation namespace

**Example:**
```bash
# List all available locales
zopio i18n --list

# Add a new locale
zopio i18n --add fr

# Create a new translation namespace
zopio i18n --create user-profile
```

### config

Manage Zopio project configuration.

```bash
zopio config [options]
```

**Options:**
- `-i, --init` - Initialize configuration file
- `-g, --get <key>` - Get configuration value
- `-s, --set <key=value>` - Set configuration value

**Example:**
```bash
zopio config --init
```

### jobs

Manage background jobs and triggers.

```bash
zopio jobs [options]
```

**Options:**
- `-l, --list` - List all registered jobs
- `-c, --create <n>` - Create a new job
- `-t, --trigger <jobId>` - Manually trigger a job
- `-s, --schedule <cron>` - Set job schedule (cron expression)

**Example:**
```bash
zopio jobs --create user-notifications
```

This command creates a new job file with internationalization support for job status messages.

### plugins

Manage CLI plugins.

```bash
zopio plugins [options]
```

**Options:**
- `-l, --list` - List all installed plugins
- `-i, --install <n>` - Install a plugin

**Example:**
```bash
zopio plugins --list
```

## CRUD Commands

### crud

Unified CRUD operations toolkit for generating resources, schemas, UI components, data providers, permissions, tests, and validation.

```bash
zopio crud <subcommand> [options]
```

#### crud resource

Generate CRUD resources with internationalization support.

```bash
zopio crud resource <n> [options]
```

**Arguments:**
- `n` - Resource name (singular, e.g., 'user', 'product')

**Options:**
- `-f, --fields <fields>` - Comma-separated list of fields with types (e.g., 'name:string,age:number')
- `-p, --path <path>` - Custom path for the resource (default: "resources")
- `-i, --i18n` - Add internationalization support (default: true)
- `--no-ui` - Skip UI component generation

**Example:**
```bash
zopio crud resource user --fields "name:string,email:string,age:number"
```

#### crud schema

Generate CRUD resources from a schema file.

```bash
zopio crud schema <schema> [options]
```

**Arguments:**
- `schema` - Path to schema file or 'sample' to generate a sample schema

**Options:**
- `-o, --output <path>` - Output directory for generated resources (default: "resources")
- `-i, --i18n` - Add internationalization support (default: true)
- `--no-ui` - Skip UI component generation

**Example:**
```bash
zopio crud schema schemas/product.json
```

#### crud ui

Generate CRUD UI templates for resources.

```bash
zopio crud ui [options]
```

**Options:**
- `-l, --list` - List available UI templates
- `-t, --template <n>` - Template to generate
- `-r, --resource <n>` - Resource name
- `-a, --all` - Generate all templates for a resource
- `-o, --output <path>` - Output directory (default: "components")
- `-i, --i18n` - Add internationalization support (default: true)
- `--typescript` - Generate TypeScript components

**Example:**
```bash
zopio crud ui --template table --resource user
```

#### crud provider

Manage data providers for CRUD operations.

```bash
zopio crud provider [options]
```

**Options:**
- `-l, --list` - List all configured data providers
- `-t, --types` - List available data provider types
- `-a, --add <n>` - Add a new data provider configuration
- `-p, --provider <type>` - Provider type (rest, graphql, firebase, etc.)
- `-o, --options <options>` - Provider options as comma-separated key=value pairs
- `-u, --update-registry` - Update the provider registry
- `-g, --generate <resource>` - Generate a CRUD resource using a specific provider
- `--provider-name <n>` - Provider name to use with generate
- `--path <path>` - Path for generated resources (default: "resources")

**Example:**
```bash
zopio crud provider --add api --provider rest
```

#### crud permissions

Manage permissions for CRUD operations.

```bash
zopio crud permissions [options]
```

**Options:**
- `-i, --init` - Initialize permissions system
- `-lr, --list-resources` - List resources with operations
- `-lo, --list-roles` - List roles with permissions
- `-ar, --add-resource <n>` - Add a resource
- `-ao, --add-operations <operations>` - Operations to add (comma-separated)
- `-rl, --add-role <n>` - Add a role
- `-rd, --role-description <description>` - Role description
- `-g, --grant` - Grant permissions to a role
- `-r, --revoke` - Revoke permissions from a role
- `-rn, --role-name <n>` - Role name for grant/revoke
- `-rs, --resource-name <n>` - Resource name for grant/revoke
- `-o, --operations <operations>` - Operations for grant/revoke (comma-separated)
- `-gp, --generate <resource>` - Generate permissions for a resource

**Example:**
```bash
zopio crud permissions --generate user
```

### crud-testing

Generate tests for CRUD operations, including API endpoint tests and UI component tests.

```bash
zopio crud-testing [options]
```

**Options:**
- `-m, --model <model>` - Model name (required)
- `-f, --fields <fields>` - Field definitions in format "name:type:example" (required)
- `-t, --test-type <type>` - Test type (api | ui | both) (default: "both")
- `-fw, --framework <framework>` - Testing framework (jest | vitest) (default: "jest")
- `-o, --output <directory>` - Output directory for tests

**Example:**
```bash
zopio crud-testing -m Product -f "name:string:Test Product,price:number:99.99,inStock:boolean:true" -t api -fw jest
```

This command generates tests for the specified model using the chosen testing framework. The field definitions include name, type, and example values used in the tests. You can generate API tests, UI component tests, or both.

### crud-validation

Generate validation schemas for models using popular validation libraries.

```bash
zopio crud-validation [options]
```

**Options:**
- `-m, --model <model>` - Model name (required)
- `-f, --fields <fields>` - Field definitions in format "name:type:validations" (required)
- `-o, --output <directory>` - Output directory for validation schemas
- `-l, --library <library>` - Validation library (yup, zod, class-validator) (default: "yup")

**Example:**
```bash
zopio crud-validation -m User -f "name:string:required|min=2,email:string:required|pattern=^[^\s@]+@[^\s@]+\.[^\s@]+$,age:number:min=18|max=120"
```

This command generates validation schemas for the specified model using the chosen validation library. The field definitions include name, type, and validation rules separated by pipes (`|`). Available validation rules include:

- `required` - Field is required
- `min=X` - Minimum value/length
- `max=X` - Maximum value/length
- `pattern=X` - Regular expression pattern

## Plugin Commands

### i18n-analyze

Analyze translation coverage across locales.

```bash
zopio i18n-analyze [options]
```

**Options:**
- `-d, --directory <directory>` - Directory containing translation files (default: "dictionaries")
- `-b, --base <locale>` - Base locale to compare against (default: "en")

**Example:**
```bash
zopio i18n-analyze --base en
```

### i18n-sync

Synchronize translation keys across locales.

```bash
zopio i18n-sync [options]
```

**Options:**
- `-d, --directory <directory>` - Directory containing translation files (default: "dictionaries")
- `-b, --base <locale>` - Base locale to sync from (default: "en")
- `-f, --file <file>` - Specific file to synchronize (optional)
- `--dry-run` - Show what would be synchronized without making changes

**Example:**
```bash
zopio i18n-sync --dry-run
```
